| 채널 | 용도 | 세션 분리 |
|------|------|---------|

## 텔레그램 설정 (5단계)

### Step 1: 봇 생성

텔레그램에서 `@BotFather`에게:

```
/newbot
이름: 내 에이전트
username: my_agent_bot
```

봇 토큰(예: `123456789:ABC-xxxxx`)을 메모합니다.

### Step 2: 토픽 그룹 만들기

1. 텔레그램에서 **그룹 생성**
2. **설정** → **Chat History** → `Visible` (슈퍼그룹으로 변환)
3. **Topics** 토글 활성화
4. 토픽 생성: `리서치`, `콘텐츠`, `분석`, `잡담` 등
5. 봇을 **관리자**로 추가 (또는 BotFather에서 `/setprivacy` → `Disable`)

### Step 3: ID 수집

- **그룹 ID**: `@userinfobot`을 그룹에 추가 → ID 확인 → 즉시 제거
- **토픽 ID**: 토픽 메시지 URL에서 `thread/xxxxx` 부분
- **내 ID**: `@userinfobot`에게 DM

### Step 4: openclaw.json 설정

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456789:ABC-xxxxx",
      "dmPolicy": "allowlist",
      "allowFrom": ["내_텔레그램_ID"],
      "groups": {
        "-1001234567890": {
          "requireMention": true,
          "topics": {
            "3": { "requireMention": false },
            "5": { "requireMention": false },
            "7": { "requireMention": false }
          }
        }
      }
    }
  }
}
```

**설정 설명:**
- `dmPolicy: "allowlist"` — 허용된 사람만 DM 가능 (보안)
- `allowFrom` — 허용할 텔레그램 사용자 ID 배열
- `requireMention: true` — 그룹 전체에서는 `@봇이름` 멘션 필요
- `requireMention: false` — 특정 토픽에서는 멘션 없이 바로 응답

### Step 5: 토픽별 에이전트 바인딩 (멀티에이전트 시)

```json
{
  "channels": {
    "telegram": {
      "groups": {
        "-1001234567890": {
          "topics": {
            "3": { "agentId": "strategy", "requireMention": false },
            "5": { "agentId": "research", "requireMention": false },
            "7": { "agentId": "content", "requireMention": false }
          }
        }
      }
    }
  }
}
```

> 토픽 3에서 말하면 전략봇이, 토픽 5에서 말하면 리서치봇이 응답합니다. 같은 그룹 안에서 **역할별 채널 분리**가 가능합니다.

## Slack 설정 (6단계)

### Step 1: Slack 앱 생성

`api.slack.com/apps` → **Create New App** → **From scratch**

### Step 2: Socket Mode 활성화

**Settings** → **Socket Mode** → Enable
App Token 생성 (이름: `openclaw`, scope: `connections:write`)
→ `xapp-...` 토큰 메모

### Step 3: OAuth 권한 설정

**OAuth & Permissions** → **Bot Token Scopes**:

```
chat:write           ← 메시지 전송
app_mentions:read    ← @멘션 감지
channels:history     ← 채널 메시지 읽기
groups:history       ← 비공개 채널 읽기
im:history           ← DM 읽기
reactions:read       ← 리액션 감지
reactions:write      ← 리액션 추가
files:read           ← 파일 읽기
```

선택 추가:
```
message.mpim         ← 그룹 DM
member_joined_channel
member_left_channel
pin_added
pin_removed
```

### Step 4: 이벤트 구독

**Event Subscriptions** → Enable → **Subscribe to bot events**:

```
app_mention          ← @멘션 이벤트 (필수)
message.channels     ← 공개 채널 메시지
message.groups       ← 비공개 채널 메시지
message.im           ← DM 메시지
reaction_added       ← 리액션 추가 (선택)
reaction_removed     ← 리액션 제거 (선택)
```

### Step 5: 워크스페이스에 설치

**Install to Workspace** → **Allow**
Bot Token (`xoxb-...`) 메모

### Step 6: openclaw.json 설정

```json
{
  "channels": {
    "slack": {
      "enabled": true,
      "mode": "socket",
      "appToken": "xapp-1-...",
      "botToken": "xoxb-...",
      "groupPolicy": "allowlist",
      "channels": {
        "C051BJG1K89": { "requireMention": true },
        "C0A2M56PKTQ": { "requireMention": false }
      }
    }
  }
}
```

채널에서 `/invite @봇이름`으로 초대하면 완료.

## 세션 분리 — 왜 중요한가?

### Slack
- `#전략` 채널 → 세션 A
- `#리서치` 채널 → 세션 B
- DM → 세션 C
- **같은 채널의 스레드** → 같은 세션 (서로 알 수 있음)

### 텔레그램
- 리서치 토픽 → `agent:main:telegram:group:-100...:topic:3`
- 콘텐츠 토픽 → `agent:main:telegram:group:-100...:topic:5`

> **세션 A에서 나눈 대화를 세션 B에서는 모릅니다.** 세션 사이의 다리 역할은 오직 **워크스페이스 파일**(MEMORY.md, memory/ 등)만 합니다.

## 실전 팁

### 1. "대화 맥락을 이어가고 싶으면 같은 채널에서"
채널을 옮기면 새 세션이 시작되어 이전 맥락을 잃습니다.

### 2. "중요한 건 '이거 메모리에 기록해줘'"
파일에 기록된 건 모든 세션에서 접근 가능합니다.

### 3. "한 세션에서 너무 많은 종류의 작업 섞지 않기"
컴팩션 시 중요한 맥락이 요약에 묻힐 수 있습니다.

### 4. "DM은 최후의 수단"
DM은 하나의 세션이라 주제가 섞이기 쉽습니다. 가능하면 토픽/채널 분리.

### 5. "새 작업 = 새 장소"
Slack이면 새 스레드, 텔레그램이면 적절한 토픽에서 시작.

## 심화: 카카오톡 연동

카카오톡 채널(비즈니스 채널)을 OpenClaw에 연결할 수 있습니다.

### 플러그인 설치

```bash
openclaw plugins install @openclaw/kakao-talkchannel
```

### openclaw.json 설정

```json5
channels: {
  kakao: {
    enabled: true,
    appKey: "${KAKAO_APP_KEY}",
    channelId: "${KAKAO_CHANNEL_ID}",
    callbackUrl: "https://your-domain.com/webhook/kakao",
    messageTypes: ["text", "textCard", "basicCard"],
    dmPolicy: "open"       // open | allowlist
  }
}
```

### 페어링 프로세스

카카오톡 채널과 OpenClaw를 연결하려면 페어링이 필요합니다.

```bash
# 페어링 코드 생성
openclaw pair kakao
# → 페어링 코드: ABCD-1234

# 카카오톡 채널 관리자센터에서
# 스마트채팅 → 웹훅 → URL 입력 → 페어링 코드 입력
```

카카오톡 채널 관리자센터에서 웹훅 URL에 `https://your-domain.com/webhook/kakao`를 입력하고, 검증 요청에 `/pair ABCD-1234`를 전송하면 연결이 완료됩니다.

### 카드 메시지 포맷

카카오톡은 일반 텍스트 외에 카드형 메시지를 지원합니다. AGENTS.md에서 에이전트에게 카드 사용 규칙을 지정할 수 있습니다.

**textCard** (텍스트 + 버튼):
```json5
{
  type: "textCard",
  title: "주문 확인",
  description: "주문번호 #1234가 접수되었습니다.",
  buttons: [
    { label: "주문 상세", action: "webLink", url: "https://..." },
    { label: "배송 조회", action: "message", text: "배송 조회 #1234" }
  ]
}
```

**basicCard** (이미지 + 텍스트 + 버튼):
```json5
{
  type: "basicCard",
  thumbnail: { url: "https://example.com/product.jpg" },
  title: "상품명",
  description: "상품 설명",
  buttons: [
    { label: "구매하기", action: "webLink", url: "https://..." },
    { label: "문의하기", action: "phone", number: "02-1234-5678" },
    { label: "공유하기", action: "share" }
  ]
}
```

### 버튼 액션 타입

| 액션 | 동작 | 용도 |
|------|------|------|
| `message` | 채팅에 텍스트 자동 입력 | 후속 질문, 명령 트리거 |
| `webLink` | 외부 URL 열기 | 상품 페이지, 주문 확인 |
| `phone` | 전화 걸기 | 고객센터 연결 |
| `share` | 카카오톡 공유 | 바이럴, 추천 |

## 심화: Tailscale 외부 접속

Gateway를 외부에서 접근하려면 Tailscale을 사용합니다. 포트 포워딩, DDNS 설정 없이 안전한 터널을 제공합니다.

### 3가지 모드

| 모드 | 접근 범위 | 인증 | 용도 |
|------|----------|------|------|
| `off` (기본) | 로컬만 (127.0.0.1) | 불필요 | 개발/테스트 |
| `serve` | Tailnet 내부만 | Tailscale 인증 (토큰리스 가능) | 팀 내부 사용 |
| `funnel` | 공개 인터넷 (HTTPS) | 비밀번호 필수 | 외부 웹훅 수신 |

### serve 설정 (Tailnet 내부)

```bash
# Tailnet 내부에서만 접근 가능한 서빙
openclaw gateway --tailscale serve

# 또는 openclaw.json에서
```

```json5
gateway: {
  bind: "tailnet",
  tailscale: {
    mode: "serve",
    hostname: "openclaw-main"    // tailnet에서의 호스트명
  }
}
```

Tailnet에 속한 기기에서 `https://openclaw-main.your-tailnet.ts.net:18789`로 접근할 수 있습니다. Tailscale 자체 인증을 사용하므로 별도 토큰 없이 접근 가능합니다.

### funnel 설정 (공개 HTTPS)

```bash
# 공개 인터넷에서 접근 가능한 HTTPS 터널
openclaw gateway --tailscale funnel
```

```json5
gateway: {
  bind: "tailnet",
  tailscale: {
    mode: "funnel",
    hostname: "openclaw-main",
    auth: { mode: "token", token: "${OPENCLAW_GATEWAY_TOKEN}" }  // 필수!
  }
}
```

> **주의**: funnel 모드에서는 `auth.token`이 **반드시 필요합니다**. 설정하지 않으면 Gateway가 시작을 거부합니다. 공개 인터넷에 토큰 없이 에이전트를 노출하면 누구나 명령을 보낼 수 있습니다.

### 지원 포트

Tailscale Funnel은 다음 포트만 지원합니다:
- **443** (기본 HTTPS)
- **8443** (대체 HTTPS)
- **10000** (커스텀)

`gateway.port`가 이 세 포트가 아니면 Tailscale이 자동으로 포트 매핑합니다 (예: 내부 18789 → 외부 443).

### CLI 명령

```bash
# serve 모드로 시작
openclaw gateway --tailscale serve

# funnel 모드로 시작
openclaw gateway --tailscale funnel

# 현재 상태 확인
openclaw gateway status
# → mode: funnel, url: https://openclaw-main.xxx.ts.net

# Tailscale 연결 해제
openclaw gateway --tailscale off
```

### launchd 데몬으로 자동 재시작

macOS에서 Gateway를 항상 실행 상태로 유지하려면 launchd를 사용합니다.

```bash
# launchd plist 자동 생성 및 등록
openclaw gateway install-daemon

# 생성되는 파일: ~/Library/LaunchAgents/com.openclaw.gateway.plist
# 자동으로: 부팅 시 시작, 크래시 시 재시작, 로그 → ~/.openclaw/logs/
```

수동으로 관리:
```bash
# 데몬 시작
launchctl load ~/Library/LaunchAgents/com.openclaw.gateway.plist

# 데몬 중지
launchctl unload ~/Library/LaunchAgents/com.openclaw.gateway.plist

# 상태 확인
launchctl list | grep openclaw
```

## 심화: 세션 분리 상세 설정

세션이 어떻게 분리되는지 세밀하게 제어할 수 있습니다.

```json5
session: {
  dmScope: "per-channel-peer",
  reset: { mode: "idle", idleMinutes: 120 },
  threadBindings: { enabled: true, idleHours: 24 }
}
```

### dmScope — DM 세션 범위

| 값 | 동작 | 예시 |
|----|------|------|
| `per-channel-peer` (기본) | 채널 + 상대방 조합으로 분리 | Slack DM 홍길동 ≠ 텔레그램 DM 홍길동 |
| `per-peer` | 상대방 기준으로 통합 | Slack DM 홍길동 = 텔레그램 DM 홍길동 (같은 세션) |

`per-peer`는 같은 사람이 여러 채널에서 말을 걸어도 하나의 세션으로 이어가고 싶을 때 사용합니다. 단, 채널 간 사용자 매핑이 USER.md에 설정되어 있어야 합니다.

```markdown
<!-- USER.md에서 채널 간 매핑 -->
## 홍길동
- Slack: U0A1B2C3D
- Telegram: 123456789
- Kakao: kakao_gildong
```

### reset — 세션 초기화 조건

| 모드 | 동작 |
|------|------|
| `idle` | 마지막 메시지 후 `idleMinutes`분 경과 시 세션 초기화 |
| `daily` | 매일 자정(KST) 기준으로 세션 초기화 |

```json5
// idle 모드: 2시간 대화 없으면 새 세션
reset: { mode: "idle", idleMinutes: 120 }

// daily 모드: 매일 자정에 세션 리셋
reset: { mode: "daily", timezone: "Asia/Seoul" }
```

세션이 초기화되면 대화 히스토리는 사라지지만, 워크스페이스 파일(MEMORY.md, memory/)은 그대로 유지됩니다. 새 세션이 시작될 때 BOOT.md가 실행되면서 메모리에서 맥락을 복구합니다.

### threadBindings — 스레드별 세션 바인딩

```json5
threadBindings: { enabled: true, idleHours: 24 }
```

`enabled: true`이면 Slack 스레드나 텔레그램 답장 체인이 **독립 세션**으로 바인딩됩니다.

| 설정 | 동작 |
|------|------|
| `enabled: true` | 스레드마다 별도 세션 (대화 주제 분리) |
| `enabled: false` | 스레드도 채널 세션에 통합 (기본 동작) |
| `idleHours: 24` | 스레드 세션이 24시간 비활성이면 자동 해제 |

> **팁**: Slack에서 한 채널에 여러 주제를 동시에 논의할 때 `threadBindings: true`를 켜면, 스레드 A에서 논의한 내용이 스레드 B에 섞이지 않습니다. 단, 스레드 간에는 워크스페이스 파일로만 정보를 전달할 수 있습니다.
