# Ch.01 워크스페이스 — 에이전트가 사는 곳

> 💡 이 챕터는 **읽기만 하면 됩니다.** 설정은 openclaw onboard가 자동으로 해줍니다.

> 🎯 **이 챕터의 목표**: 에이전트의 파일 구조를 이해하는 것. 아직 아무것도 만들 필요 없습니다. 읽기만 하세요.

> "에이전트는 코드가 아니라 **집**이 필요합니다."

---

## 에이전트 = 두뇌 + 몸

| 구성 | 역할 | 예시 |
|------|------|------|
| **두뇌** | AI 모델 (언어 이해, 판단, 생성) | Claude Sonnet 4.6, GPT-5 |
| **몸** | OpenClaw (도구, 채널, 스케줄링, 메모리) | 메시지 전달, API 호출, 파일 관리 |

두뇌만 있으면 "생각만 하는 AI". 몸만 있으면 "빈 로봇". 둘이 합쳐져야 **일하는 에이전트**가 됩니다.

**쉽게 말하면**: ChatGPT에 "손과 발"을 달아준 게 OpenClaw입니다. 생각만 하던 AI가 이제 직접 메시지를 보내고, 파일을 만들고, 스케줄에 맞춰 일합니다.

---

## 설치

```bash
npm install -g openclaw@latest

# 초기 설정 마법사
openclaw onboard
```

`openclaw onboard`가 안내하는 것:
1. AI 모델 선택 (Claude / OpenAI / 로컬 모델)
2. API 키 입력
3. 채널 연결 (텔레그램, Slack 등)
4. 워크스페이스 생성

## 워크스페이스 디렉토리 구조

> 💡 이 구조는 `openclaw onboard`가 **자동으로 만들어줍니다**. 외울 필요 없고, "아 이런 파일들이 있구나" 정도만 알면 됩니다.

설치가 끝나면 `~/.openclaw/` 아래에 에이전트의 "집"이 만들어집니다.

```
~/.openclaw/
├── openclaw.json          ← 🏠 집 설계도 (Gateway 설정)
├── .env                   ← 🔑 열쇠 (API 키, 봇 토큰)
├── AGENTS.md              ← 📋 업무 매뉴얼 (행동 규칙, 절대 규칙)
│
├── workspace-main/        ← 에이전트 1의 방
│   ├── SOUL.md            ← 🧬 DNA (정체성, 성격, 말투)
│   ├── IDENTITY.md        ← 🪪 신분증 (이름, 아바타, 프로필)
│   ├── USER.md            ← 👥 팀원 정보 (이름, 역할, 권한)
│   ├── TOOLS.md           ← 🔧 도구 설명서 (API, SDK 접속 정보)
│   ├── HEARTBEAT.md       ← ⏰ 알람시계 (자동 점검 루틴)
│   ├── MEMORY.md          ← 🧠 장기 기억 (핵심 참조 정보)
│   ├── memory/            ← 📔 일일 기록
│   │   ├── 2026-03-20.md
│   │   ├── 2026-03-21.md
│   │   └── 2026-03-22.md
│   └── skills/            ← ⚡ 스킬 폴더
│       └── my-skill/
│           ├── SKILL.md
│           └── scripts/
│
├── workspace-second/      ← 에이전트 2의 방 (같은 구조)
│   ├── SOUL.md
│   └── ...
│
├── agents/                ← 세션 저장소
├── cron/                  ← 크론잡 설정
└── logs/                  ← 로그
```

## 핵심 파일 7개 — 한 눈에 보기

| 파일 | 비유 | 역할 | 자동 로딩 |
|------|------|------|----------|
| **SOUL.md** | DNA | 성격, 말투, 역할 정의 | ✅ 매 세션 |
| **AGENTS.md** | 업무 매뉴얼 | 행동 규칙, 절대 규칙, 파일 규칙 | ✅ 매 세션 |
| **IDENTITY.md** | 신분증 | 이름, 품종(?), 아바타 정보 | ✅ 매 세션 |
| **USER.md** | 팀원 명부 | 팀원 이름, Slack ID, 역할, 권한 | ✅ 매 세션 |
| **TOOLS.md** | 도구 설명서 | API 키 위치, SDK 실행법, 접속 정보 | ✅ 매 세션 |
| **HEARTBEAT.md** | 알람시계 | 매 시간 자동 체크리스트 | ✅ heartbeat 시 |
| **MEMORY.md** | 장기 기억 | 핵심 URL, 테이블 구조, 진행 중 TODO | 세션 시작 시 읽음 |

> **핵심 포인트**: SOUL.md ~ TOOLS.md 5개 파일은 매 대화마다 **자동으로** 시스템 프롬프트에 주입됩니다. 수동으로 "읽어줘"라고 할 필요가 없습니다. 이것이 터미널 Claude Code(CLAUDE.md 1개만 자동 로딩)와의 가장 큰 차이입니다.

## 세션 — 대화 한 판

세션은 에이전트와 나누는 **대화 한 묶음**입니다.

```
[메시지 도착]
   ↓
OpenClaw가 시스템 프롬프트 조립
(SOUL.md + AGENTS.md + TOOLS.md + USER.md + IDENTITY.md)
   ↓
+ 이 세션의 대화 히스토리 (압축된 과거 + 최근 원본)
   ↓
+ 방금 들어온 메시지
   ↓
= 전체 컨텍스트 → API 호출 → 응답
```

### 세션 분리 원칙

- **Slack**: 채널마다 별도 세션. 스레드는 같은 세션.
- **텔레그램**: 채팅방/토픽마다 별도 세션.
- **DM**: 사람마다 별도 세션.

```
┌─────────────────────────────────────┐
│          워크스페이스 (파일)            │
│  AGENTS.md  SOUL.md  memory/        │
│  TOOLS.md   MEMORY.md  ...          │
│                                     │
│  ┌──────┐  ┌──────┐  ┌──────┐      │
│  │세션 A │  │세션 B │  │세션 C │      │
│  │Slack  │  │Slack  │  │텔레그램│      │
│  │#채널1 │  │DM     │  │그룹   │      │
│  └──────┘  └──────┘  └──────┘      │
│                                     │
│  대화 맥락: 각각 독립                  │
│  파일: 전부 공유                      │
└─────────────────────────────────────┘
```

> **대화 맥락**은 채널별로 분리되지만, **워크스페이스 파일**(AGENTS.md, memory/ 등)은 모든 세션이 공유합니다.

## 컴팩션 — 대화가 길어지면?

대화가 길어져서 컨텍스트 윈도우의 70%를 넘기면 **자동 압축(compaction)** 발동:

```
[턴 1~50] → 컨텍스트 윈도우 70% 도달
   ↓
[컴팩션 발동] → 턴 1~30을 요약으로 압축
   ↓
[요약 + 턴 31~50 원본] → 계속 대화 가능
   ↓
[턴 51~80] → 또 70% 도달 → 다시 컴팩션
   ↓
... 반복
```

**압축되지 않는 것**: 시스템 프롬프트(SOUL.md, AGENTS.md 등)는 항상 원본 그대로.
**압축되는 것**: 오래된 대화 내용.

> **팁**: 중요한 정보는 항상 "이거 메모리에 기록해줘"라고 요청하세요. 파일에 적힌 건 컴팩션과 무관하게 생존합니다.

## 프로덕션 openclaw.json 전체 레퍼런스

실서비스 환경에서 사용하는 전체 설정입니다. 모든 키를 다 채울 필요는 없습니다. 기본값이 있으므로 변경이 필요한 부분만 오버라이드하면 됩니다.

```json5
{
  gateway: {
    port: 18789,
    bind: "loopback",       // loopback | tailnet | 0.0.0.0
    auth: { mode: "token", token: "${OPENCLAW_GATEWAY_TOKEN}" },
    channelHealthCheckMinutes: 5,
    channelStaleEventThresholdMinutes: 30,
    channelMaxRestartsPerHour: 10
  },
  agents: {
    defaults: {
      model: { primary: "anthropic/claude-sonnet-4-6", fallbacks: ["openai/gpt-5.2"] },
      bootstrapMaxChars: 20000,
      bootstrapTotalMaxChars: 150000,
      heartbeat: { every: "30m", target: "last", lightContext: false },
      sandbox: { mode: "non-main", scope: "agent" },
      contextPruning: { mode: "cache-ttl", ttl: "1h" },
      compaction: { mode: "safeguard" }
    },
    list: [
      { id: "main", default: true, workspace: "~/.openclaw/workspace-main" },
      { id: "external", workspace: "~/.openclaw/workspace-external" }
    ]
  },
  tools: {
    profile: "messaging",      // messaging | coding | full
    deny: ["group:automation"],
    agentToAgent: { enabled: true, allow: ["main","external"] },
    exec: { security: "allowlist", ask: "on-miss" },
    fs: { workspaceOnly: true },
    elevated: { enabled: false }
  },
  session: {
    dmScope: "per-channel-peer",
    reset: { mode: "idle", idleMinutes: 120 },
    threadBindings: { enabled: true, idleHours: 24 }
  },
  cron: { enabled: true, maxConcurrentRuns: 2, sessionRetention: "24h" },
  hooks: {
    enabled: true,
    token: "64-char-hex-token",
    mappings: [{ match: { path: "channeltalk" }, action: "agent", agentId: "external" }]
  },
  skills: {
    entries: { "daily-briefing": { enabled: true } },
    load: { watch: true }
  },
  logging: { redactSensitive: "tools", redactPatterns: ["api[_-]?key","secret","token","password"] }
}
```

### 주요 섹션 설명

| 섹션 | 역할 |
|------|------|
| `gateway` | 포트, 바인딩, 인증, 채널 헬스체크 주기 |
| `agents.defaults` | 모든 에이전트에 적용되는 기본값 (모델, 부트스트랩 제한, 하트비트, 샌드박스) |
| `agents.list` | 에이전트 목록과 워크스페이스 경로 |
| `tools` | 도구 프로파일, 실행 권한, 파일시스템 제한 |
| `session` | DM 범위, 세션 초기화, 스레드 바인딩 |
| `cron` | 크론잡 활성화, 동시 실행 제한 |
| `hooks` | 외부 웹훅 수신 및 에이전트 라우팅 |
| `skills` | 스킬 활성화 및 핫 로딩 |
| `logging` | 민감 정보 마스킹 패턴 |

### Config 관리 CLI 명령어

```bash
# 현재 설정 조회 (점 표기법으로 경로 지정)
openclaw config get agents.defaults
openclaw config get gateway.port

# 설정 변경 (자동으로 openclaw.json 업데이트)
openclaw config set gateway.port 18790
openclaw config set agents.defaults.model.primary "anthropic/claude-sonnet-4-6"

# 전체 설정 검증 — 문제 발견 시 해결 방법까지 안내
openclaw doctor

# 자동 수정 모드 — 고칠 수 있는 것은 바로 수정
openclaw doctor --fix
```

`openclaw doctor`가 체크하는 항목:
- API 키 유효성
- 채널 연결 상태
- 모델 접근 권한
- 워크스페이스 디렉토리 존재 여부
- 부트스트랩 파일 크기 제한 초과 여부
- 포트 충돌

### Hot Reload 모드

`openclaw.json`을 수정하면 Gateway를 재시작해야 할까요? **설정에 따라 다릅니다.**

```json5
gateway: {
  configReload: "hybrid"   // hybrid | hot | restart | off
}
```

| 모드 | 동작 | 용도 |
|------|------|------|
| `hybrid` (기본) | 안전한 키만 핫 리로드, 위험한 키는 재시작 필요 | 프로덕션 권장 |
| `hot` | 모든 변경을 즉시 반영 | 개발/테스트 환경 |
| `restart` | 어떤 변경이든 Gateway 재시작 필요 | 최대 안정성 |
| `off` | 파일 감시 안 함 | CI/CD 파이프라인 |

**핫 리로드 가능한 키**: `agents.defaults.model`, `tools.deny`, `session.reset`, `logging.*`
**재시작 필요한 키**: `gateway.port`, `gateway.bind`, `channels.*`

변경 시 로그:
```
[gateway] config reload: agents.defaults.model → hot-reloaded
[gateway] config reload: gateway.port → restart required (run: openclaw gateway restart)
```

### $include로 설정 분할

설정 파일이 커지면 `$include`로 분리할 수 있습니다.

```json5
// openclaw.json
{
  gateway: { "$include": "./config/gateway.json" },
  agents: { "$include": "./config/agents.json" },
  channels: { "$include": "./config/channels.json" },
  tools: { "$include": "./config/tools.json" },
  hooks: { "$include": "./config/hooks.json" }
}
```

```
~/.openclaw/
├── openclaw.json              ← 메인 (각 섹션을 $include로 참조)
├── config/
│   ├── gateway.json
│   ├── agents.json
│   ├── channels.json
│   ├── tools.json
│   └── hooks.json
```

규칙:
- `$include` 경로는 `openclaw.json` 기준 상대 경로
- 중첩 `$include`는 지원하지 않음 (1단계만)
- `openclaw doctor`가 분할 파일도 함께 검증

### 부트스트랩 파일 전체 목록

워크스페이스 안에 들어가는 모든 파일과 역할입니다.

| 파일/폴더 | 로딩 시점 | 역할 | 필수 여부 |
|-----------|----------|------|----------|
| **AGENTS.md** | 매 세션 | 행동 규칙, 절대 규칙, 권한 체계 | 권장 |
| **SOUL.md** | 매 세션 | 정체성, 성격, 말투, 경계 | 권장 |
| **USER.md** | 매 세션 | 팀원 정보 (이름, ID, 역할, 권한) | 선택 |
| **IDENTITY.md** | 매 세션 | 에이전트 이름, 이모지, 아바타, 한줄 설명 | 선택 |
| **TOOLS.md** | 매 세션 | API 키 위치, SDK 접속 정보, 도구 가이드 | 선택 |
| **HEARTBEAT.md** | heartbeat 시 | 자동 점검 체크리스트 | 선택 |
| **BOOT.md** | 매 세션 시작 시 | 세션 시작 시 실행할 체크리스트 (매번 실행) | 선택 |
| **BOOTSTRAP.md** | 최초 1회 | 워크스페이스 초기화 루틴 (한 번만 실행) | 선택 |
| **MEMORY.md** | 세션 시작 시 읽음 | 장기 기억: 핵심 URL, DB 구조, 진행 중 TODO | 권장 |
| **memory/** | 세션 시작 시 최근 N일 | 일일 기록 (YYYY-MM-DD.md) | 선택 |
| **skills/** | 스킬 호출 시 | 스킬 정의 폴더 (SKILL.md + scripts/) | 선택 |

> **부트스트랩 크기 제한**: 개별 파일 최대 `bootstrapMaxChars`(기본 20,000자), 전체 합계 최대 `bootstrapTotalMaxChars`(기본 150,000자). 초과하면 파일 끝이 잘립니다. `openclaw doctor`로 초과 여부를 확인하세요.

## 에이전트의 하루

```
새벽   하트비트 깨움 → 할 일 체크 → 없으면 HEARTBEAT_OK → 대기
08:00  아침 브리핑 → 오늘 할 일 정리 → 채널에 전송
낮     채널 메시지 도착 → 요청 처리 (검색, 분석, 파일 작업)
수시   웹훅 도착 → 자동 처리 (CS 답변, 알림 전달)
21:00  이브닝 리뷰 → 오늘 완료 건 정리 → 메모리에 기록
밤     하트비트로 순찰 → 긴급건만 처리 → 대기
```

## 실습: 첫 에이전트 만들기

### Step 1: 설치 확인

```bash
openclaw --version
# 2026.2.26 이상이면 OK
```

### Step 2: 워크스페이스 확인

```bash
ls ~/.openclaw/
# openclaw.json, workspace-main/ 등이 보이면 OK
```

```bash
openclaw doctor
```

```bash
openclaw gateway
```
