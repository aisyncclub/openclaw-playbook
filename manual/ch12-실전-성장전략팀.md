# Ch.12 실전: 성장 전략팀 5인 구축

지금까지 OpenClaw의 개념과 도구를 배웠습니다. 이제 진짜 써먹을 차례입니다.

이 챕터에서는 **성장 전략팀 5인**을 처음부터 끝까지 구축합니다. 복사-붙여넣기만으로 돌아가는 실전 설정을 제공하니, 따라 하시면 됩니다.

---

## 왜 성장 전략팀인가

혼자서 커머스, 유튜브, SaaS를 동시에 운영하면 어떻게 될까요?

- 커머스 월 1.5억 매출 관리
- 유튜브 채널 콘텐츠 기획 및 제작
- SaaS 제품 개발과 마케팅

**사람 한 명이 하면**: 하루 18시간 일해도 모자랍니다.
**AI 팀 5인이 하면**: 각자 전문 영역을 24시간 커버합니다.

핵심은 **레버리지**입니다. 내 시간 1을 투입해서 결과물 10을 얻는 구조를 만드는 것.

---

## 레버리지 체인

이 팀이 만드는 가치 흐름은 다음과 같습니다:

```
시간 →(리서치봇)→ 지식 →(콘텐츠봇)→ 콘텐츠 →(퍼널봇)→ 수익
                                                    ↑
데이터 →(분석봇)→ 인사이트 →(전략봇)→ 의사결정 ────┘
```

**위쪽 라인**: 리서치봇이 시간을 지식으로 바꾸고, 콘텐츠봇이 지식을 콘텐츠로 바꾸고, 퍼널봇이 콘텐츠를 수익으로 바꿉니다.

**아래쪽 라인**: 분석봇이 데이터를 인사이트로 바꾸고, 전략봇이 인사이트를 의사결정으로 바꿉니다. 이 의사결정이 다시 퍼널봇의 수익 최적화에 반영됩니다.

두 라인이 합쳐져서 **자기 강화 루프**가 완성됩니다.

## 5인 팀 구성

| 에이전트 | 역할 | 모델 | Temperature | Heartbeat |
|---------|------|------|-------------|-----------|
| **전략봇** (Strategy) | 3개 사업 통합 전략, 우선순위 결정, 주간 브리핑 | claude-sonnet-4-20250514 | 0.3 | 매주 월요일 09:00 |
| **리서치봇** (Research) | 시장 조사, 경쟁사 분석, AI 도구 평가 | claude-sonnet-4-20250514 | 0.4 | 매일 08:00 |
| **콘텐츠봇** (Content) | 유튜브 스크립트, 블로그, SNS 콘텐츠 생성 | claude-sonnet-4-20250514 | 0.7 | 매일 10:00 |
| **퍼널봇** (Funnel) | 리드 마그넷 설계, 전환 최적화, 매출 추적 | claude-sonnet-4-20250514 | 0.3 | 매일 14:00 |
| **분석봇** (Analytics) | 매출 분석, KPI 추적, 이상치 감지, P&L | claude-sonnet-4-20250514 | 0.2 | 매일 07:00 |

**Temperature 설정 기준**:
- 0.2~0.3: 정확성이 중요한 분석/전략 업무
- 0.4: 리서치 (사실 기반이되 연결 능력 필요)
- 0.7: 콘텐츠 (창의성 필요)

## AGENTS.md 라우팅 규칙

AGENTS.md는 OpenClaw가 사용자 메시지를 어느 에이전트에게 보낼지 결정하는 규칙입니다.

```markdown
# 에이전트 라우팅 규칙

## 키워드 → 에이전트 매핑

| 키워드 | 담당 에이전트 | 설명 |
|--------|-------------|------|
| 전략, 우선순위, 브리핑, 방향, 계획 | strategy | 사업 전략과 의사결정 |
| 조사, 리서치, 경쟁사, 트렌드, 도구 | research | 시장 조사와 정보 수집 |
| 영상, 스크립트, 유튜브, 콘텐츠, 썸네일 | content | 콘텐츠 기획과 제작 |
| 퍼널, 전환, 리드, 마케팅, 랜딩 | funnel | 퍼널 설계와 전환 최적화 |
| 매출, 분석, KPI, 수치, 비용, ROI | analytics | 데이터 분석과 리포팅 |

## 기본값
키워드가 매칭되지 않으면 → **strategy** (전략봇이 판단 후 적절한 에이전트에게 위임)

## 복합 요청
여러 키워드가 섞인 경우 → **strategy**가 받아서 분배
예: "유튜브 매출 분석해줘" → strategy → analytics + content
```

## 에이전트 간 통신 규칙

에이전트들은 `sessions_send`를 통해 서로 메시지를 주고받습니다. 하지만 아무나 아무에게나 보내면 혼란스러워집니다. 규칙이 필요합니다.

### 통신 흐름도

```
리서치봇 ──→ 콘텐츠봇    (조사 결과를 콘텐츠 소재로)
리서치봇 ──→ 전략봇      (시장 변화를 전략 수립에)
분석봇  ──→ 전략봇      (데이터 인사이트를 의사결정에)
분석봇  ──→ 퍼널봇      (전환율 데이터를 최적화에)
전략봇  ──→ 전체        (의사결정을 전 에이전트에)
퍼널봇  ──→ 콘텐츠봇    (CTA 요구사항을 콘텐츠에)
```

### 메시지 포맷 규칙

에이전트 간 메시지는 반드시 다음 포맷을 따릅니다:

```
[보내는봇] → [받는봇]
유형: 요청/보고/긴급
내용: (본문)
기대 액션: (받는봇이 해야 할 일)
```

**예시**:
```
[리서치봇] → [콘텐츠봇]
유형: 보고
내용: 드랩 AI 상세페이지 도구 분석 완료. 10분 내 전문가 수준 상세페이지 생성 가능.
      경쟁사 대비 가격 30% 저렴. 셀러 후기 평점 4.7/5.
기대 액션: "AI 상세페이지 도구 비교" 유튜브 스크립트에 드랩 섹션 추가
```

```markdown
## 3개 사업 통합 관리

| 사업 | 현재 단계 | 월 목표 |
|------|----------|---------|
| 커머스 | 운영 안정화 | 매출 1.5억 유지 + 자동화 확대 |

## 우선순위 매트릭스

모든 의사결정은 **Impact x Urgency** 매트릭스로 판단합니다:

|          | 긴급 (Urgent) | 비긴급 (Not Urgent) |
|----------|--------------|-------------------|
| **고영향** | 즉시 실행 (DO) | 일정 잡기 (SCHEDULE) |
| **저영향** | 위임 (DELEGATE) | 제거 (ELIMINATE) |

- Impact 기준: 매출 기여도 (40%), 성장 기여도 (30%), 시스템화 기여도 (30%)
- Urgency 기준: 마감일, 시장 타이밍, 의존성

- 기회: {내용}

**4. 에이전트별 이번주 미션**
```

```markdown
## 스크립트 구조

모든 스크립트는 다음 5단계를 따릅니다:

### 1. 후크 (0:00-0:30)
- 충격적인 수치 또는 질문으로 시작
- "AI 없이 8시간 걸리던 작업을 10분 만에 끝냈습니다"
- 시청자가 스크롤을 멈추게 만드는 한 문장

### 2. 문제 제기 (0:30-2:00)
- 시청자가 겪고 있는 고통 묘사
- 공감 포인트: "이거 저만 그런 거 아니죠?"
- 기존 방법의 한계 지적

### 3. 해결책 제시 (2:00-8:00)
- AI 도구/방법론 소개
- 실제 화면 시연 (Before/After)
- 구체적 수치 제공 (시간, 비용, 매출)

### 4. 심화 (8:00-12:00)
- 프로 팁, 주의사항, 고급 활용법
- "대부분 여기서 멈추는데, 한 단계 더 가면..."
- 전문성을 보여주는 구간

### 5. CTA (12:00-끝)
- 구체적 행동 유도
- 리드 마그넷 연결 ("설명란에 무료 체크리스트 링크")
- 다음 영상 예고
```

```markdown
## 전환율 공식

```
매출 = 트래픽 x 리드 전환율 x 구매 전환율 x 객단가

예시:
- 유튜브 월 조회수: 100,000
- 리드 전환율: 2% → 2,000 리드
- 구매 전환율: 3% → 60명 구매
- 평균 객단가: 40만원
- 월 매출 = 2,400만원
```

최적화 순서: 객단가 → 구매 전환율 → 리드 전환율 → 트래픽
(뒤에서부터 높이는 것이 ROI가 가장 높습니다)
```

```markdown
## ROI 공식

각 활동의 ROI를 다음과 같이 계산합니다:

```
ROI = (수익 - 비용) / 비용 x 100%

예시 — 유튜브 교육 퍼널:
- 영상 제작 비용: 50만원 (시간 환산)
- 광고비: 30만원
- 총 비용: 80만원
- 교육 매출: 400만원
- ROI = (400 - 80) / 80 x 100% = 400%
```

ROI 100% 미만 활동은 전략봇에 보고하여 중단/개선 판단을 요청합니다.
```

## 배포 방법

### 방법 1: 로컬 직접 실행 (가장 간단)

```bash
# 1. 워크스페이스 생성
mkdir -p ~/.openclaw/workspace-{strategy,research,content,funnel,analysis}

# 2. 설정 파일 복사 (위의 openclaw.json을 ~/.openclaw/openclaw.json에 저장)
cp openclaw.json ~/.openclaw/openclaw.json

# 3. 각 에이전트 SOUL.md, HEARTBEAT.md를 워크스페이스에 복사
# (이 매뉴얼의 .openclaw/ 폴더 전체를 ~/.openclaw/에 복사)

# 4. .env 파일에 API 키와 봇 토큰 설정
cp .env.example ~/.openclaw/.env
# → .env 파일을 열고 실제 값 입력

# 5. Gateway 시작
openclaw gateway

# 6. 상태 확인
openclaw doctor
```

### 방법 2: 데몬으로 상시 실행 (Mac)

```bash
# Gateway를 macOS 데몬으로 등록 (부팅 시 자동 시작, 크래시 시 자동 재시작)
openclaw gateway install-daemon

# 상태 확인
launchctl list | grep openclaw

# 중지
openclaw gateway stop
```

### 방법 3: 클라우드 서버 (VPS)

```bash
# GCP, AWS, DigitalOcean 등 Linux 서버에서:
npm install -g openclaw@latest
openclaw onboard --install-daemon

# Tailscale로 원격 접속 (Ch.04 심화 참조)
openclaw gateway --tailscale serve
```

> **Docker를 쓰고 싶다면**: OpenClaw 공식 Docker 이미지 `ghcr.io/openclaw/openclaw:latest`를 사용하세요. `docker run -v ~/.openclaw:/root/.openclaw -p 18790:18790 ghcr.io/openclaw/openclaw:latest gateway`

### 핵심 프로덕션 설정 포인트

| 설정 | 목적 |
|------|------|
| `agents.defaults.model.fallbacks` | Sonnet 실패 시 Haiku로 자동 전환, 가용성 확보 + 비용 절감 |
| `agents.defaults.cacheRetention: "long"` | 시스템 프롬프트 캐시로 반복 비용 90% 절감 |
| `agents.defaults.contextPruning` | 100K 토큰 도달 시 자동 요약 압축 |
| `tools.agentToAgent.allow` | 허용된 통신 경로만 열어 혼란 방지 |
| `tools.exec.allowlist` | 허용된 4개 명령만 실행 가능, 나머지 전부 차단 |
| `tools.fs.workspaceOnly` | 프로젝트 디렉토리 밖 파일 접근 차단 |
| `cron.maxConcurrentRuns: 3` | heartbeat 동시 실행을 3개로 제한하여 API 비용 폭주 방지 |
| `logging.redactPatterns` | 로그에서 민감 정보 자동 마스킹 |
