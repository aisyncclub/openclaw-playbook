## 1단계: API Key 발급

OpenClaw는 AI 모델의 "두뇌"를 빌려 쓰는 구조입니다. 두뇌를 빌리려면 **API Key**(접속 열쇠)가 필요합니다.

### Anthropic (Claude) API Key — 권장

1. [console.anthropic.com](https://console.anthropic.com) 접속
2. **Sign Up** 클릭 → 이메일로 회원가입 (Google 계정 연동 가능)
3. 로그인 후 왼쪽 메뉴에서 **API Keys** 클릭
4. **Create Key** 버튼 클릭
5. 이름을 입력하고 (예: `openclaw`) **Create** 클릭
6. `sk-ant-api01-xxxxx...` 형태의 키가 표시됩니다 — **이 순간에만 볼 수 있으므로 반드시 복사해서 안전한 곳에 저장**

> 무료 크레딧이 제공되지 않는 경우, Settings > Billing에서 결제 수단을 등록해야 API 호출이 가능합니다.

### OpenAI (GPT) API Key — 선택

1. [platform.openai.com](https://platform.openai.com) 접속
2. **Sign Up** 또는 **Log In**
3. 왼쪽 메뉴에서 **API Keys** 클릭
4. **Create new secret key** 클릭
5. `sk-proj-xxxxx...` 형태의 키가 표시됩니다 — 마찬가지로 이 순간에만 볼 수 있습니다

> Claude와 GPT 중 하나만 있어도 됩니다. 처음이라면 Claude를 권장합니다.

### OAuth 방식 — API Key 없이 구독으로 사용하기

API Key 대신 **기존 유료 구독**을 그대로 활용하는 방법입니다. 종량제가 아닌 **정액제**라 비용 예측이 쉽습니다.

#### OpenAI Codex Pro OAuth (권장, $200/월 정액)

ChatGPT Plus/Pro 구독이 있으면 그대로 사용 가능합니다.

```bash
# 방법 1: onboard 시 선택
openclaw onboard --auth-choice openai-codex

# 방법 2: 이미 설치된 경우
openclaw models auth login --provider openai-codex
# → 브라우저가 열림 → OpenAI 로그인 → 승인
```

브라우저에서 OpenAI 로그인 후 승인하면 자동으로 토큰이 저장됩니다.

**원격 서버(헤드리스)에서 설정하는 경우:**
브라우저 콜백(`http://127.0.0.1:1455`)이 실패하면, 브라우저 주소창의 URL을 복사해서 터미널에 직접 붙여넣으세요.

#### Claude OAuth — 현재 차단됨

> **2026년 1월 9일부터 Anthropic이 서드파티 도구에서의 OAuth 사용을 차단했습니다.**

Claude Pro/Max 구독의 OAuth 토큰은 OpenClaw에서 더 이상 사용할 수 없습니다. Claude를 쓰려면 반드시 **API Key (종량제)**를 사용하세요.

| 인증 방식 | 비용 | OpenClaw 사용 |
|----------|------|:------------:|
| OpenAI Codex Pro OAuth | $200/월 정액 | ✅ 가능 |
| Claude Pro/Max OAuth | $20~200/월 정액 | ❌ 차단됨 |
| Claude API Key | 종량제 (토큰당) | ✅ 가능 |
| OpenAI API Key | 종량제 (토큰당) | ✅ 가능 |

#### 인증 상태 확인

```bash
openclaw models status
# → 현재 연결된 모델, 인증 방식, 만료 여부 표시
```

### Ollama 로컬 모델 — 완전 무료 ($0)

API Key도 OAuth도 없이, 내 컴퓨터에서 AI 모델을 직접 돌릴 수 있습니다.

**Ollama 설치:**
```bash
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.ai/install.sh | sh

# Windows
# ollama.ai에서 설치 파일 다운로드
```

**OpenClaw와 연동:**
```bash
# 가장 간단한 방법 — Ollama가 OpenClaw를 자동 설치/설정
ollama launch openclaw --model qwen3:8b

# 또는 수동 설정
openclaw onboard
# → 모델 선택에서 "Ollama/GPT OSS" 선택
```

**권장 모델:**

| 모델 | VRAM | 용도 |
|------|------|------|
| `qwen3:8b` | 8GB | 노트북에 적합, 범용 |
| `llama3.2:3b` | 4GB | Heartbeat 전용 (최소 비용) |
| `kimi-k2.5:cloud` | 클라우드 | 로컬 자원 부족 시 |

> **요구사항**: 최소 64K 토큰 컨텍스트 길이, 16GB+ VRAM 권장. M1 Mac은 8GB에서도 3b 모델 구동 가능.

**비용 비교:**

| 방식 | 월 비용 | 특징 |
|------|---------|------|
| Ollama 로컬 | $0 (전기세만) | 느림, 프라이버시 최고 |
| Claude API | $10-70 | 빠름, 고품질 |
| Codex Pro OAuth | $200 정액 | 무제한, 빠름 |

### OpenRouter — 여러 AI 모델을 하나로

OpenRouter는 Claude, GPT, Gemini, Mistral 등 **100+ 모델을 하나의 API Key**로 사용할 수 있는 서비스입니다.

```bash
openclaw onboard
# → 모델 선택에서 OpenRouter 선택
# → OpenRouter API Key 입력 (openrouter.ai에서 발급)
```

**openclaw.json 설정:**
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "openrouter/anthropic/claude-sonnet-4-6",
        "fallbacks": ["openrouter/openai/gpt-5", "openrouter/google/gemini-2.0-flash"]
      }
    }
  }
}
```

**자동 모델 선택:**
```json
"primary": "openrouter/openrouter/auto"
```
`auto`를 지정하면 프롬프트에 따라 최적 모델을 자동 선택합니다.

> **보안**: `openclaw onboard`로 설정하면 API Key가 시스템 키체인에 저장되어 config 파일에 노출되지 않습니다.

## 3단계: 설치

### Mac 설치 (권장)

1. **터미널 열기**: `Cmd + Space` → "Terminal" 검색 → 실행
2. **Node.js 설치** (npm이 함께 설치됩니다):

```bash
# Homebrew가 있다면:
brew install node

# 없다면 Node.js 공식 사이트에서 다운로드:
# https://nodejs.org → LTS 버전 다운로드 → 설치
```

3. **OpenClaw 설치**:

```bash
npm install -g openclaw@latest
```

> `npm`은 Node.js를 설치하면 자동으로 함께 설치됩니다. "npm: command not found" 에러가 나면 Node.js 설치가 안 된 것입니다.

### Windows 설치

Windows에서는 3가지 방법이 있습니다. **WSL2를 가장 권장**합니다.

#### 방법 1: WSL2 (Windows Subsystem for Linux) — 권장

**PowerShell을 관리자 모드로 실행** 후:

```powershell
# WSL2 + Ubuntu 설치
wsl --install -d Ubuntu-24.04
# 재부팅 후 Ubuntu 터미널이 자동으로 열림
# 사용자 이름과 비밀번호 설정
```

Ubuntu 터미널에서:

```bash
# systemd 활성화 (자동 실행에 필요)
sudo tee /etc/wsl.conf >/dev/null <<'EOF'
[boot]
systemd=true
EOF
```

PowerShell에서 WSL 재시작:

```powershell
wsl --shutdown
```

다시 Ubuntu 터미널에서 OpenClaw 설치:

```bash
npm install -g openclaw@latest
openclaw onboard
```

**Gateway 자동 시작 설정:**

```bash
# 로그인 없이 서비스 실행 허용
sudo loginctl enable-linger "$(whoami)"

# Gateway 서비스 등록
openclaw gateway install
```

#### 방법 2: Docker Desktop

Docker Desktop이 설치되어 있다면:

```bash
git clone https://github.com/OpenClawAI/openclaw.git
cd openclaw

# .env 파일 생성
echo "ANTHROPIC_API_KEY=your_key_here" > .env
echo "TELEGRAM_BOT_TOKEN=your_bot_token" >> .env

# 빌드 및 실행
docker compose build
docker compose run --rm opencli onboard
docker compose up -d gateway
```

#### 방법 3: 네이티브 Windows (제한적)

PowerShell에서:

```powershell
# 빠른 설치
iwr -useb https://openclaw.ai/install.ps1 | iex

# 설정
openclaw onboard

# 실행
openclaw gateway run
```

> **주의**: 네이티브 Windows는 일부 기능이 제한됩니다 (Tailscale 연동, 일부 플러그인). 안정적인 운영을 원하면 WSL2를 사용하세요.

### Linux 설치

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

## 핵심 용어 5분 설명

OpenClaw를 쓰면서 자주 나오는 용어들입니다. 지금 외울 필요 없고, 나중에 헷갈릴 때 다시 보면 됩니다.

| 용어 | 한 줄 설명 | 비유 |
|------|-----------|------|
| **API Key** | AI 서비스에 접속하는 비밀 열쇠 | 건물 출입카드 |
| **JSON** | 설정 파일 형식. 중괄호 `{}`로 묶인 키-값 쌍 | 이력서 양식 (항목: 내용) |
| **토큰** | AI가 처리하는 텍스트 단위. 한글 1글자 ≈ 2~3토큰 | 택시 미터기의 "100m" 단위. 비용의 기본 단위 |
| **Gateway** | 에이전트의 통신 관제탑. 메시지를 받아서 적절한 에이전트에게 전달 | 공항 관제탑 |
| **세션** | 대화 한 묶음. 채팅방 하나 = 세션 하나 | 카카오톡 채팅방 1개 |
| **시스템 프롬프트** | 에이전트가 매번 읽는 기본 지침서. "너는 ~한 역할이야"라고 알려주는 문서 | 신입 사원의 업무 매뉴얼 |
| **웹훅** | "이런 일이 생기면 이 URL로 알려줘"라는 자동 알림 설정 | 택배 도착 알림 문자 |
| **컨텍스트 윈도우** | 에이전트가 한 번에 기억할 수 있는 대화 크기 | 책상 위 공간. 넓을수록 많은 자료를 동시에 펼쳐볼 수 있음 |

## 처음 5분 테스트

설정이 끝났으면 바로 테스트합니다. 목표: **텔레그램에서 "안녕" 보내기 → AI 응답 받기**.

### 1. Gateway 시작

```bash
openclaw gateway
```

터미널에 `🐾 Gateway listening on :18790` 메시지가 뜨면 성공입니다.

### 2. 텔레그램에서 메시지 보내기

1. 텔레그램 앱을 열고, 아까 만든 봇을 검색합니다
2. `/start` 를 먼저 보냅니다 (봇 활성화)
3. **"안녕"** 이라고 보냅니다

### 3. 응답 확인

몇 초 안에 AI가 응답합니다. 응답이 오면 **설정 완료**입니다.

> 응답이 안 온다면:
> - 터미널에서 `openclaw doctor`를 다시 실행해서 빨간 항목이 있는지 확인
> - API Key가 올바른지 확인 (오타, 앞뒤 공백)
> - 텔레그램 봇 토큰이 올바른지 확인
> - Gateway가 실행 중인지 확인 (터미널 창이 닫히지 않았는지)
