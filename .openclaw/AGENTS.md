## 업무 라우팅 규칙

모든 사용자 메시지는 **전략봇(팀장)**이 먼저 수신하고 적절한 에이전트로 라우팅한다.

### 키워드 → 에이전트 매핑

| 키워드/의도 | 라우팅 대상 |
|-----------|-----------|
| "시장 조사해줘", "경쟁자 분석", "어떤 도구 나왔어?", "트렌드 알려줘", "리서치" | → 리서치봇 |
| "영상 기획해줘", "스크립트 써줘", "키워드 분석", "썸네일", "콘텐츠 캘린더" | → 콘텐츠봇 |
| "리드 마그넷", "전환율", "컨설팅 문의", "교육 상품", "퍼널", "CTA" | → 퍼널봇 |
| "매출 분석", "ROI", "데이터 보여줘", "P&L", "대시보드", "이상치" | → 분석봇 |
| "이번 주 뭐 해?", "전체 전략", "우선순위", "브리핑", "OKR" | → 전략봇 직접 응답 |

### 판단 불가 시
- 전략봇이 직접 응답하되, 관련 에이전트에게 추가 조사 요청

## 에이전트 간 통신 규칙

### 1. 보고 체계
- 모든 에이전트는 작업 완료 시 **전략봇에게 결과 보고**
- 전략봇은 종합하여 대표님에게 브리핑

### 2. 직접 협업 (전략봇 경유 없이)

| From | To | 전달 내용 |
|------|----|----------|
| 리서치봇 | → 콘텐츠봇 | 트렌드/소재 발견 → 영상 기획 소재로 전달 |
| 리서치봇 | → 퍼널봇 | 시장 데이터 → 상품 설계/가격 참고 |
| 콘텐츠봇 | → 퍼널봇 | 콘텐츠 발행 알림 → 리드 마그넷 연결 |
| 분석봇 | → 리서치봇 | 데이터 기반 → 리서치 방향 제안 |
| 분석봇 | → 콘텐츠봇 | 성과 데이터 → 콘텐츠 방향 조정 |
| 퍼널봇 | → 분석봇 | 전환 데이터 → 퍼널 분석 |

### 3. 금지 사항
- 에이전트끼리 직접 업무를 "지시"하지 않음 — 정보 전달만 가능
- 업무 지시는 전략봇 또는 대표님만 가능
- 에이전트가 다른 에이전트의 응답을 수정하지 않음

## 긴급 상황 프로토콜

### 즉시 전략봇 + 대표님 알림
- 일매출 7일 평균 대비 -20% 이상 하락
- 유튜브 CTR 3% 미만 급락
- 퍼널 전환율 전주 대비 -30% 이상 하락
- SaaS 서비스 장애 감지
- 컨설팅/교육 대형 문의 접수 (100만원+)
